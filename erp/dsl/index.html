Entity


Entity is something that will be stored in the database. 

entity Company

Here Company is the name of the entity.

It is convesional to start an entity name for a capital letter and user a singular form of the word. To user several words, use upper CamelCase:

  • Company - good
  • InvoiceItem - good
  • company - bad
  • Companies - bad
  • Invoice_Item - bad

Fields


entity Company {
  field code: Int
  field name: String
}

Two fields are declared here: code and name. The code is integer and name is string. Convensionally, lower CamelCase is used for field names:

  • name - good
  • itemQuantity - good
  • Name - bad
  • Companies - bad
  • item_Quantity - bad

You can set a default value for a field: field name: String = "Default Name". If you have a default value, you can omit the data type declaration: field name = "Default Name". This field will be considered String automatically because "Default Name" is String.

Compute Fields


Compute fields are not stored in the database but they are visible to the user. To declare a compute property, use the compute keyword instead of field and set an expression for calculation. 

entity Invoice {
  field qty: Dec[2]
  compute price: Dec[2] = total/qty
  field total: Dec[2]
}

Here we declare the compute field price. You can simplify the declaration and omit the datatype. It will be infered automatically: field price = total/qty.

This compute field will be read only. If you want to make it editable, yuo have to use the update modifier: 

compute price = total/qty {
  update {
    total = qty*price
  }
}

This means that when the price field is changed, we will update total with qty*price.

Reference Fields


A reference field contains reference to another or the same entity. To do this, you can easily use the name of the entity as data type after colon: 

entity Invoice {
  field company: Company
}

entity Company {
  field name: String
  
  func toString = name
}

You can make references to any entities but you have to define a string representation for the entity because the reference field will use the string representation to show the reference to user. To do this, you need to declare toString function and define an expression how to convert the entity to string.

Enum Fields


If you have a fixed set of variants to choose, it is recommended to use an enum field. 

enum TransactionType {
  case Invoice
  case Sale {
    label = "Sale Label"
  }
}

entity Transaction {
  field type: TransactionType
}

TransactionType is an enum class, and you can see field type is an enum field. TransactionType contains to cases Invoice and Sale. This field will be displayed to use as a combo box and user will be able to select an option either Invoice or Sale. You can use label modifier as usual to provide a text for user. Convensionally, cases should use upper CamelCase, so they should be started from Capital letter.

Collection Fields


Collection fields allow to add many items of similar type into an entity. 

entity Invoice {
  field items: [InvoiceItem]
}

class InvoiceItem {
  field product: Product
  field qty: Dec[2]
}

InvoiceItem is a class. This means that it can be stored in the database only as a part of another entity. It does not have an id and cannot be selected itself, but only with another entity which will use it.

items is a collection field because the datatype declared in square brackets items. This means that there are many items. This field won't be visible in list view, and will be represented as a table in form by default.

Inheritance


trait HasCode {
  field code: String
}
trait ActionItem {
  func name: String
  field priority: Int
  field status: Status
}

entity Product extends HasCode {
  field description: String
}
entity Company extends HasCode with ActionItem {
  field name: String
}

Traits can be a base for entities and classes. Entity can inherit multiple traits. All fields declared in the traits will be available in the entity. Product will have code field declared in the HasCode trait. It declares alse description field. So it will have these two fields.

In this example the Company entity will have code field from HasCode trait and priority and status from ActionItem. Field name is declared in the Company. In ActionItem name is declared as func. This means that the trait itself does not create a field but require form entity to provide either a field, compute field or a function with the same signature. This is more flexible and allows to use existing fields to calculate value and satisfy the trait's requirenments.

It allows to avoid duplication and reuse the same structure in similar entities. It also allows to define a contract which entities should satisfy when implementing this trait.

Extensions


entity Company {
  field name: String
}
extend entity Company {
  field description: String
}

You can add field into existing entity using the extend entity construction. In this example, Company will have the description field as well as name. It allows to add user-specific fields into existing applications.

Syntax


Class ::=
[extend] {entity | class | trait | enum} <Name> [extends <SuperClass>*] {
  {<Field> | <ComputeField> | <Function> | <View> | <Case>}*
}

Field ::= field <Name>: [<DataType>] [= <Expression>] [{ <Modifier>* }]
ComputeField ::= compute <Name>: [<DataType>] = <Expression> [{ <Modifier>* }]
Function ::= func <Name>[(<Param>*)]: [<DataType>] = <Expression>
View ::= view <Name> = <Layout>
Case ::= case <Name> [{ <Modifier>* }]

Modifier ::= <ModifierName> {[= <Expression>] | [{ <Expression>* }] }

View


Introduction


Entities always contain two view. list is to display many items and form is to display one item. If these views are not defined in an entity, they will be generated automatically. You can use also different names for your views. 

entity Company {
  field code: Int
  field name: String
  
  view list = table{code, name}
}

| code | name     |
-------------------
| 1    | Company1 |
| 2    | Company2 |

To declare a view, you should start it with keyword view. Then you can use any name for view and after = you can define layout.

is a type of layout. In braces there sublayouts. In this case there are two fields code and name. They will be displayed as table columns. 

entity Invoice {
  field number: String
  field data: Date
  field items: [InvoiceItem]
  view form = column {
    net {
      number
      date
    }
    items => table(selectable){product, qty}
  }
}
class InvoiceItem {
  field product: Product
  field qty: Dec[2]
}

number:| numberField  | 
  date:| dateField    |

| product  | qty |
------------------
| Product1 | 10  |
| Product2 | 23  |

This is more complex example. column puts sublayouts one under another vertically. net create a layout with labels and fields as a form. Every field will have its label before it. All fields will be alligned.

items => means that table will show items with columns product and qty. After layout type in brackets there could be modifiers. In this example table(selectable), selectable is a modifier which allows to table to select rows.

Field


Field layout displays fields. When you use an expression as a layout by default the field layout will be used. This means that you need to use this layout explicitly only if you need to provide some modifiers. 

entity Company {
  field code: Int
  field name: String
  field description: String
  
  view form = net {
    row {code, name}
    description => field(span = 2)
  }
}

|       code:| codeField |  name:| nameField |
|description:| descriptionField              |

Field modifiers

span: Int
Allows to expand the field on more than one column in parent layout. Default value is 1.

Universal modifiers


Some modifiers can be used for all layouts.

label: String
Set label for layout which will be displayed in the header. By default, entity name will be used. It will be automatically translated into user-friendly string. For lists, singualar form of entity name will be translated into plural. It does not work properly for all cases. So if plural for is incorrect, provide the label modifier. 
view list = table(label = "Companies") {code, name}

Language


Operators


=
Set operator changes value of the left operand to the value of right operand: price = total/qty

Math operators

+
Add two numeric values 1 + 2 == 3.
-
1 - 2 == -1.
*
2*3 == 6.

Comparison operators

==
Equal operator returns true if two values are equal. 1 == 1 == true
!=
Not equal operator returns true if two values are not equal. 1 != 1 == false
>
Greater operator. 2 > 1
>=
Greater or equal operator. 2 >= 1
<
Less operator. 1 < 2
<=
Less or equal operator. 1 <= 2

Logical operators

!
Inverse unary logical operator returns true if the operand is false and visa versa. !true == false, !false == true
&&
Logical and operator returns true only if both operands are true otherwise false. true && true == true, true && false == false
||
Logical or operator returns false only if both operands are false otherwise true. false || false == false, true || false == true

Order

Binary operators have a natural order and you can use brackets to control the order: 1 + 2*3 == 1 + (2*3) != (1 + 2)*3 If operators are in the same group the order will be from left to right.
  1. =
  2. ||
  3. &&
  4. ==, !=, >, >=, <, <=
  5. +, -
  6. *, /

Data Types


Dec[precision]

Decimal datatype. You have to set precision in square brackets. The expression can be a constant like field foo: Dec[2] or an expression field foo: Dec[currency.precision].

Int

Integer datatype. It is similar to Dec[0].

String

String datatype.

Text

Formatted text. Multiline string editor will be used for fields with this type.

Date, Time and DateTime

Three datatypes for date, time and date&time together

Option T?

This time contains either one item or nothing.

isEmpty: Bool
Returns whether the option contains an item or nothing.
get: T
Get the value of optional item. If it is empty, computation will fail.

Collection [T]

Collection data type of any items.

first: T?, last: T?
Optional first or last element of the collection. If the collection is empty this will return the empty value. 
[2, 3, 1].first == 2
[2, 3, 1].last == 1
[].first.isEmpty
ui selected: T?
Function returns currently selected value on UI or None. This function can be used only on UI. See also selectable modifier.
